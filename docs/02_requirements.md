### FR-03. 무한매수법 V2.2 — 매수 가이드

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-03-1 | 현재 회차가 전반전(1~20)인지 후반전(21~40)인지 판별한다. | 필수 |
| FR-03-2 | **전반전** 매수 가이드를 계산하여 표시한다. | 필수 |
| | • 작은수 LOC: 가격 = A(평단가, 1회차는 C), 수량 = (U×0.5)/가격 | |
| | • 큰수 LOC: 가격 = C×1.15, 수량 = (U×0.5)/(C×1.15) | |
| FR-03-3 | **후반전** 매수 가이드를 계산하여 표시한다. | 필수 |
| | • 조건 확인: A > C (평단가 > 전일 종가) 일 때만 주문 | |
| | • 조건 충족: 평단 LOC, 가격 = A, 수량 = U/A | |
| | • 조건 불충족: "오늘은 매수 없음" 안내 | |
| FR-03-4 | 1 unit(U)을 하루에 초과 소진하지 않도록 금액 합계를 검증한다. | 필수 |
| FR-03-5 | 수량은 소수점 둘째 자리 이하를 내림 처리한다. | 필수 |
| FR-03-6 | 40회차를 모두 소진한 경우 영혼의 LOC 안내를 표시한다. | 필수 |
| FR-03-7 | 전반전 → 후반전 전환 시점(21회차 첫날) 안내 메시지를 표시한다. | 필수 |

### FR-07. 밸류 리밸런싱 (VR) — 리밸런싱 가이드

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-07-1 | 이번 주기 목표값(V)과 현금 잔고(C)를 함께 표시한다. | 필수 |
| FR-07-2 | 현재 평가금액(E = 보유수량 × 현재가)을 자동 계산하여 표시한다. | 필수 |
| FR-07-3 | 밴드 이탈 여부를 판정한다. | 필수 |
| | • 상단 이탈: E > V × (1 + band_upper) → 매도 가이드 | |
| | • 하단 이탈: E < V × (1 - band_lower) → 매수 가이드 | |
| | • 밴드 내: "유지" 메시지 | |
| FR-07-4 | 매수 가이드: 필요 매수금액과 현금 잔고(C) 중 적은 값으로 제한하여 표시한다. | 필수 |
| FR-07-5 | 현금 부족으로 매수 금액이 제한되는 경우 "현금 부족" 경고를 표시한다. | 필수 |
| FR-07-6 | 리밸런싱 체결 입력 후 다음 주기 목표값 V를 자동 갱신한다. | 필수 |
| | • V(다음) = V(현재) × (1 + G), G = 주당 0.5% | |
| FR-07-7 | 리밸런싱 없이 주기가 경과한 경우에도 V를 갱신할 수 있다. | 필수 |
| FR-07-8 | VR 현금 잔고(C)를 수동으로 입금/출금 조정할 수 있다. | 필수 |

## 2. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01. 정확성

| ID | 요구사항 |
|----|---------|
| NFR-01-1 | 평단가 계산은 부동소수점 오류를 최소화하기 위해 `Decimal` 타입을 사용한다. |
| NFR-01-2 | 수량 내림 처리는 `math.floor`를 사용하며, 소수 둘째 자리(0.01주 단위)까지 허용한다. |
| NFR-01-3 | 동일한 입력에 대해 항상 동일한 계산 결과를 반환한다. (재현성) |
| NFR-01-4 | 전략 엔진 함수는 부작용(side effect) 없는 순수 함수로 구현한다. |

### NFR-02. 데이터 영속성

| ID | 요구사항 |
|----|---------|
| NFR-02-1 | 모든 체결 기록은 로컬 SQLite 파일(`data/jkhenry.db`)에 영구 저장한다. |
| NFR-02-2 | 평단가·보유수량·회차는 체결 기록(trade)에서 매번 파생 계산하며, 단일 진실 원천(single source of truth)을 유지한다. |
| NFR-02-3 | 앱 재시작 후에도 모든 데이터가 유지된다. |
| NFR-02-4 | DB 파일을 직접 복사하여 백업할 수 있는 구조로 유지한다. |

### NFR-03. 가용성

| ID | 요구사항 |
|----|---------|
| NFR-03-1 | 인터넷 연결이 없어도 시세 조회 기능을 제외한 모든 기능이 동작한다. |
| NFR-03-2 | 시세 조회 실패 시 앱이 크래시 되지 않고 graceful degradation을 제공한다. |

### NFR-04. 성능

| ID | 요구사항 |
|----|---------|
| NFR-04-1 | 가이드 계산 화면 로드는 시세 조회 제외 1초 이내 완료한다. |
| NFR-04-2 | 체결 기록 입력 후 화면 갱신은 2초 이내 완료한다. |
| NFR-04-3 | 세션 내 동일 티커 시세는 캐싱하여 중복 API 호출을 방지한다. |

### NFR-05. 사용성

| ID | 요구사항 |
|----|---------|
| NFR-05-1 | 브라우저 주소창에 `http://localhost:8501` 입력만으로 접속 가능하다. |
| NFR-05-2 | 모든 화면은 PC 웹 브라우저에서 스크롤 없이 핵심 정보가 보이도록 구성한다. |

## 3. 데이터 요구사항 (Data Requirements)

### 3-1. 핵심 관리 변수

> 어떤 변수가 **저장(DB)**되고, 어떤 변수가 **파생 계산**되는지 명확히 구분한다.

#### 무한매수법 V2.2

| 변수 | 기호 | 저장 방식 | 계산 방법 |
|------|------|---------|---------|
| 총 시드 | seed | DB 저장 (포트폴리오) | 사용자 입력 |
| 분할 횟수 | T | DB 저장 (파라미터) | 기본값 45 |
| 1회 매수금 | U | **파생** | seed / T |
| 전일 종가 | C | 세션 캐시 | yfinance 조회 |
| 누적 매수금 | — | **파생** | Σ(체결 수량 × 체결 단가) — 매수 체결만 |
| 누적 매수 수량 | — | **파생** | Σ(체결 수량) — 매수 체결만 |
| 현재 평단가 | A | **파생** | 누적 매수금 / 누적 매수 수량 |
| 보유 수량 | — | **파생** | 매수 수량 합계 − 매도 수량 합계 |
| 현재 회차 | — | **파생** | 매수 체결이 있는 고유 날짜 수 (또는 매수 체결 건수) |
| 전반전/후반전 | — | **파생** | 회차 ≤ 20 이면 전반전, 21~40 이면 후반전 |
| 사이클 상태 | — | DB 저장 (사이클) | active / completed / loss_cut |

#### 밸류 리밸런싱 (VR)

| 변수 | 기호 | 저장 방식 | 계산 방법 |
|------|------|---------|---------|
| 현재 주기 목표값 | V | DB 저장 (포트폴리오) | 사용자 초기 설정 후 주기마다 갱신 |
| 현금 잔고 | C | DB 저장 (포트폴리오) | 매수 시 감소, 매도 시 증가, 수동 조정 가능 |
| 주기당 성장률 | G | DB 저장 (파라미터) | 기본값 0.005 (주당 0.5%) |
| 상단 밴드 폭 | band_upper | DB 저장 (파라미터) | 기본값 0.20 (20%) |
| 하단 밴드 폭 | band_lower | DB 저장 (파라미터) | 기본값 0.20 (20%) |
| 보유 수량 | — | **파생** | trade 기록에서 계산 |
| 현재 평가금액 | E | **파생** | 보유 수량 × 현재가 |
| 상단 밴드 | — | **파생** | V × (1 + band_upper) |
| 하단 밴드 | — | **파생** | V × (1 - band_lower) |

### 3-2. 엔티티 목록

| 엔티티 | 설명 | 주요 필드 |
|--------|------|---------|
| **Portfolio** | 운용 단위 | id, name, ticker, strategy(IB/VR), seed_capital, params(JSON), status, created_at |
| **Cycle** | IB 사이클 단위 | id, portfolio_id, start_date, end_date, status(active/completed/loss_cut), profit, profit_rate |
| **Trade** | 체결 기록 (단일 진실 원천) | id, portfolio_id, cycle_id, trade_date, side(BUY/SELL), order_type(LOC/LIMIT/MARKET), shares, price, amount |
| **VrPeriod** | VR 주기 기록 | id, portfolio_id, period_start, period_end, v_target, cash_balance, action(BUY/SELL/HOLD), e_value |

### 3-3. 파라미터 기본값 (config)

```python
# 무한매수법 V2.2
IB_DEFAULT = {
    "T": 45,                    # 분할 횟수
    "half_round": 20,           # 전반전 경계 회차
    "buy_high_multiplier": 1.15, # 큰수 LOC 가격 = 전일 종가 × 1.15
    "sell_target_pct": 0.10,    # 매도 목표: 평단 +10%
}

# 밸류 리밸런싱
VR_DEFAULT = {
    "G": 0.005,         # 주당 성장률 0.5%
    "band_upper": 0.20, # 상단 밴드 폭 20%
    "band_lower": 0.20, # 하단 밴드 폭 20%
    "period_days": 7,   # 리밸런싱 주기 7일
}
```
