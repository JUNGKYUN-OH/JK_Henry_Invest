## 2. 일일 사용 흐름 (User Journey)

### 2-1. 평상시 (무한매수법 V2.2 진행 중)

```
[장 시작 전 / 오전]
1. 브라우저에서 앱 접속 (http://localhost:8501)
2. 대시보드에서 운용 중인 포트폴리오 목록 확인
   → 각 종목의 현재 회차, 평단가, 보유수량, 미실현 손익 표시
3. 원하는 포트폴리오 선택 → "오늘의 매수 가이드" 화면
4. 앱이 yfinance로 전일 종가(C)를 자동 조회
5. 오늘 주문할 가격·수량·주문 유형을 계산해서 표시
   예시 (전반전, 평단가 A=$60.00, 전일 종가 C=$61.20, U=$100):
   ┌───────────────┬───────────────┬────────┬──────────┬──────────┐
   │ 주문 구분      │ 주문 가격      │ 수량   │ 금액(USD)│ 주문유형  │
   ├───────────────┼───────────────┼────────┼──────────┼──────────┤
   │ 작은수 LOC 매수│ $60.00 (평단) │ 0.83주 │ $50.00   │ LOC      │
   │ 큰수 LOC 매수  │ $70.38 (종가  │ 0.71주 │ $50.00   │ LOC      │
   │               │ ×1.15)        │        │          │          │
   └───────────────┴───────────────┴────────┴──────────┴──────────┘
   총 오늘 투입금: $100 (= 1 unit)

[장 중 / 오후]
6. 사용자가 증권사 앱에서 직접 LOC 주문 실행

[장 마감 후]
7. 체결 여부 확인 → 앱에서 "체결 기록 입력" 화면
8. 체결된 주문 수량·가격 입력 (미체결 시 0 입력 또는 건너뜀)
9. 앱이 평단가·보유수량·진행 회차 자동 갱신
10. 업데이트된 포트폴리오 상태 확인 후 종료
```

### 2-2. 매도 상시 주문 (평단+10% LOC)

```
[매일 장 시작 전]
- 앱이 "오늘 매도 LOC 주문" 가격·수량 표시
- 가격: 평단가 × 1.10 (항상 전량 걸어둠)
- 체결 시: 체결 기록 입력 → 보유수량 0으로 갱신 → 사이클 종료 처리
```

### 2-3. 40회차 소진 후 처리

```
- 40회차 모두 매수 완료 후 목표가 미달인 경우
- 앱이 "영혼의 LOC 매도" 안내 표시
  → 보유 전량을 평단+10% LOC로 계속 걸어둠
  → 체결될 때까지 매일 안내 유지
- 사용자 판단으로 손절 또는 홀드 결정
```

### 2-4. VR (밸류 리밸런싱) 운용

```
[매주 토요일 오전 — 미국 금요일 장 마감 후]
1. VR 포트폴리오 화면 접속
2. 앱이 yfinance로 금요일 종가 자동 조회
3. 현재 평가금액 = 보유수량 × 종가 → 자동 계산
4. 이번 주기 목표값 V와 현금 잔고 C 함께 표시
5. 밴드 이탈 여부 판정 및 가이드:
   - 상단 초과: 매도 가이드 (현금 한도 내 POOL 경계까지 매도)
   - 하단 미달: 매수 가이드 (현금 잔고 C 한도 내 POOL 경계까지 매수)
   - 밴드 내: "유지" (주문 불필요)
6. 가이드에 따라 증권사 앱 주문 실행 후 체결 기록 입력
7. 다음 주기 목표값 V 자동 갱신: V(다음) = V(현재) × (1 + G)
```

### 4-1. 무한매수법 V2.2 규칙 명세

#### 기본 설정

| 파라미터 | 기호 | 확정값 | 설명 |
|---------|------|--------|------|
| 분할 횟수 | T | **45** | 총 시드를 45번에 나눠 매수 (버퍼 5회 포함) |
| 전반전 범위 | - | **1 ~ 20회차** | 1회차부터 20회차까지 전반전 |
| 후반전 범위 | - | **21 ~ 40회차** | 21회차부터 40회차까지 후반전 (잔여 5회는 버퍼) |
| 1회 매수금 | U | **총 시드 / 45** | 매 회차 기본 매수 단위 금액 |
| 전일 종가 | C | yfinance 자동 조회 | 매수 가격 계산 기준 |
| 현재 평단가 | A | 체결 기록에서 파생 계산 | 누적 매수금 / 누적 매수 수량 |

#### 매수 규칙 (공식 계산식 포함)

**전반전 (1회차 ~ 20회차) — 매일 1 unit 소진, 0.5 unit × 2회 주문**

| 주문 구분 | 주문 가격 | 수량 계산식 | 금액 | 주문 유형 |
|---------|---------|-----------|------|---------|
| 작은수 LOC | A (현재 평단가) | (U × 0.5) / A | 0.5 × U | LOC |
| 큰수 LOC | C × 1.15 (전일 종가 +15%) | (U × 0.5) / (C × 1.15) | 0.5 × U | LOC |

> - 큰수 가격 = 전일 종가 × 1.15 → LOC 주문이므로 당일 종가 이하라면 어디서든 체결
> - 1회차에는 평단가가 없으므로 작은수 LOC 가격 = C (전일 종가) 사용
> - 소수점 수량은 소수 둘째 자리 내림 처리 (증권사마다 소수점 주문 정책 다름)

**후반전 (21회차 ~ 40회차) — 조건부 1 unit 올인**

| 조건 | 주문 구분 | 주문 가격 | 수량 계산식 | 금액 | 주문 유형 |
|------|---------|---------|-----------|------|---------|
| **A > C (평단 > 전일 종가)** | 후반전 LOC | A (평단가) | U / A | 1 × U | LOC |
| A ≤ C (평단 ≤ 전일 종가) | 매수 없음 | — | — | — | — |

> - 후반전 진입 조건: 평단가가 전일 종가보다 높은 경우에만 매수 주문
> - 평단가가 이미 전일 종가 이하라면 오늘은 매수 건너뜀

#### 매도 규칙 — 상시 LOC 매도 주문 (매일 유지)

| 파라미터 | 확정값 | 설명 |
|---------|--------|------|
| 매도 목표가 | A × 1.10 (평단가 +10%) | 매일 이 가격으로 LOC 주문 유지 |
| 매도 수량 | 보유 수량 **전량 (100%)** | 분할 매도 없음, 한 번에 전량 |
| 주문 유형 | LOC | 매일 장 마감 30분 전 제출 |

> - 체결되면: 사이클 종료 → 수익 확정 → 다음 사이클 자동 시작
> - 미체결이면: 다음날 갱신된 평단가 기준으로 재계산하여 재주문

#### 40회차 소진 후 처리 (영혼의 LOC)

| 상황 | 처리 방식 |
|------|----------|
| 40회차 매수 완료, 목표가 미도달 | 매도 LOC (평단+10%)를 체결될 때까지 매일 계속 유지 |
| 사용자 판단 손절 | 수동으로 시장가 손절, 사이클 종료(손절) 처리 |

> V2.2 공식에서는 쿼터 손절 조항 없음. 40회차 소진 후에도 매도 LOC를 유지하는 것이 정석.

#### 회차 카운트 규칙

| 항목 | 규칙 |
|------|------|
| 회차 증가 시점 | 매수 주문 체결이 확인된 날 +1 |
| 미체결일 | 회차 증가 없음, 동일 회차 유지 |
| 부분 체결 | 체결된 수량만 평단가에 반영, 회차는 +1 |

#### 사이클 흐름도

```
[새 사이클 시작 — 1회차]
   ↓
[전반전: 1회차 ~ 20회차]
  매일: 작은수 LOC (0.5U, 평단가) + 큰수 LOC (0.5U, 종가×1.15)
  동시에: 매도 LOC (전량, 평단×1.10) 상시 유지
  체결 시마다: 평단가·보유수량·회차 갱신
   ↓
[후반전: 21회차 ~ 40회차]
  매일: 평단 > 전일 종가 조건 확인
    → 조건 충족 시: LOC 1 unit (평단가)
    → 조건 불충족 시: 매수 없음
  동시에: 매도 LOC (전량, 평단×1.10) 상시 유지
   ↓
[매도 LOC 체결] → 사이클 종료 (수익)
  수익률 확정 → 다음 사이클 자동 시작
   ↓   또는
[40회차 소진] → 영혼의 LOC 유지 (매도 조건 달성 시까지)
  사용자 판단 시 수동 손절 가능
```

### 4-2. 밸류 리밸런싱 (VR) 규칙 명세

#### 필수 관리 지표 (동시 트래킹)

| 지표 | 기호 | 설명 |
|------|------|------|
| 이번 주기 목표 평가금액 | V | 현재 주기에 보유 자산이 달성해야 할 목표 금액 |
| 현금 잔고 | C | VR 운용에 배정된 현금 계좌 잔액 (매수/매도 시 증감) |
| 현재 평가금액 | E | 보유 수량 × 현재가 (yfinance 조회) |

#### 기본 설정

| 파라미터 | 기호 | 확정값 | 설명 |
|---------|------|--------|------|
| 초기 목표값 | V₀ | 사용자 설정 | 최초 투자금 (= 주식 평가금액 + 현금) |
| 주기당 성장률 | G | **주당 0.5%** (월 약 2%) | 매 주기 종료 시 V에 곱하는 성장률 |
| 리밸런싱 주기 | — | **주 1회 (토요일 오전)** | 미국 금요일 종가 기준으로 체크 |

#### 밴드 설정 및 판정

| 파라미터 | 기호 | 확정값 | 설명 |
|---------|------|--------|------|
| 상단 밴드 | — | V × (1 + band_upper) | 이 이상이면 매도 가이드 |
| 하단 밴드 | — | V × (1 - band_lower) | 이 이하이면 매수 가이드 |
| 상단 밴드 폭 | band_upper | 사용자 설정 (예: 20%) | 조정 가능한 파라미터 |
| 하단 밴드 폭 | band_lower | 사용자 설정 (예: 20%) | 조정 가능한 파라미터 |

#### VR 리밸런싱 로직

| 상황 | 조건 | 행동 | 제약 |
|------|------|------|------|
| 상단 초과 | E > V × (1 + band_upper) | 매도 가이드: E를 POOL 상단까지 줄이도록 매도 | 매도 수량 = (E - V) / 현재가 만큼 (현금 한도 역방향) |
| 하단 미달 | E < V × (1 - band_lower) | 매수 가이드: E를 POOL 하단까지 채우도록 매수 | 매수 금액 ≤ 현금 잔고 C |
| 밴드 내 | 하단 ≤ E ≤ 상단 | 유지 (리밸런싱 없음) | — |

> **POOL 경계**: 단순히 E를 V로 되돌리는 것이 아니라, 현금 잔고(C) 한도 내에서만
> 매수/매도를 실행. 현금이 부족하면 매수 가능 금액을 C로 제한하여 안내.

#### 목표값 갱신

```
리밸런싱 체크 후 (매수/매도 여부 무관):
  V(다음 주기) = V(현재 주기) × (1 + G)
  → 다음 주 토요일에 사용할 새 목표값으로 저장
```

#### VR 흐름도

```
[매주 토요일 오전]
   ↓
[금요일 종가 자동 조회 (yfinance)]
   ↓
[현재 평가금액 E = 보유수량 × 금요일 종가]
[목표값 V, 현금 잔고 C 함께 표시]
   ↓
[밴드 판정]
  ├─ E > 상단 밴드: 매도 가이드 → 증권사 앱 매도 → 체결 기록 입력 → C 증가
  ├─ E < 하단 밴드: 매수 가이드 → 증권사 앱 매수 → 체결 기록 입력 → C 감소
  └─ 밴드 내: 유지 메시지 표시
   ↓
[V 갱신: V(다음) = V(현재) × (1 + G)]
[다음 주 토요일까지 대기]
```

## 5. 경계 케이스 및 예외 상황

| 상황 | 처리 방안 |
|------|----------|
| yfinance 시세 조회 실패 | 마지막 저장 값 표시 + 경고 배너, 수동 가격 입력 허용 |

| 40회차 완료 후 목표가 미달 | 영혼의 LOC 매도 안내 지속, 회차 카운트 동결 |

| 여러 종목 동시 운용 | 각 포트폴리오 독립 처리, 대시보드에서 요약 |
